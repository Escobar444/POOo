1. IMPORTACIONES Y CONFIGURACIÓN

import readline from "readline";

const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout
});

¿Qué es esto?

  • readline es una librería que permite hacer preguntas en la terminal y recibir respuestas del usuario
  • createInterface() crea una "interfaz" para poder hacer preguntas
  • input: process.stdin = entrada del usuario desde el teclado
  • output: process.stdout = salida que se muestra en pantalla
Analogy: Es como crear un "mostrador" en una tienda donde puedes hacer preguntas (preguntas a través de la terminal).

2. CLASE ABSTRACTA PACIENTE

abstract class Paciente {
    protected nombre: string;
    protected fechaNacimiento: string;
    protected edad: number;
    protected telefono: number;
    protected responsable: string;
    protected telefonoResponsable: number;

¿Qué significa cada cosa?

abstract class Es una "plantilla" que no se puede usar directamente, solo se heredan sus propiedades y métodos
protected Las propiedades solo pueden ser accedidas por esta clase y sus subclases (herencia)
string Tipo de dato = texto (nombre, fechaNacimiento, responsable)
number Tipo de dato = números (edad, telefono)

Constructor

constructor(nombre: string, fechaNacimiento: string, edad: number, 
             telefono: number, responsable: string, telefonoResponsable: number) {
    this.nombre = nombre;
    this.fechaNacimiento = fechaNacimiento;
    this.edad = edad;
    // ... etc
}
  • El constructor se ejecuta cuando creas un nuevo objeto
  • this.nombre = nombre guarda el valor recibido en la propiedad de la clase
  • Recibe 6 parámetros (los datos del paciente)

Método Abstracto

abstract pasarConsulta(sintomas: string): void;
  • abstract = Método que DEBE ser implementado en las subclases (es obligatorio)
  • (sintomas: string) = recibe los síntomas como parámetro de texto
  • void = No retorna nada, solo ejecuta acciones

Método Mostrar Datos

mostrarDatos(): void {
    console.log("Nombre: " + this.nombre);
    console.log("Edad: " + this.edad);
    // ... etc
}
  • Imprime todos los datos guardados del paciente
  • this.nombre = accede a la propiedad nombre de esta clase
  • El + concatena (une) texto: "Nombre: " + "Juan" = "Nombre: Juan"

3. TRES CLASES HEREDADAS

ConsultaGeneral extends Paciente

class ConsultaGeneral extends Paciente {
    pasarConsulta(sintomas: string): void {
        this.mostrarDatos();
        const costo = 50;
        const descuento = 14;
        const descuentoCalculado = (costo * descuento) / 100;  // 50 * 14 / 100 = $7
        const costoFinal = costo - descuentoCalculado;         // 50 - 7 = $43
        
        console.log("Síntomas: " + sintomas);
        console.log("Descuento (14%): $" + descuentoCalculado.toFixed(2));
        console.log("COSTO A CANCELAR: $" + costoFinal.toFixed(2));
    }
}
  • extends Paciente = hereda todas las propiedades y métodos de Paciente
  • Implementa el método abstracto pasarConsulta()
  • Calcula: $50 menos el 14% de descuento = $43
  • toFixed(2) = muestra solo 2 decimales (ejemplo: 7.00)

ConsultaEspecialidad extends Paciente

class ConsultaEspecialidad extends Paciente {
    pasarConsulta(sintomas: string): void {
        this.mostrarDatos();
        const costo = 90;
        const descuento = 14;
        const descuentoCalculado = (costo * descuento) / 100;      // 90 * 14 / 100 = $12.6
        const costoConDescuento = costo - descuentoCalculado;      // 90 - 12.6 = $77.4
        const descuentoEquipo = (costoConDescuento * 3) / 100;     // 77.4 * 3 / 100 = $2.32
        const costoFinal = costoConDescuento - descuentoEquipo;    // 77.4 - 2.32 = $75.08
    }
}
  • Costo base: $90
  • Primer descuento: 14% del costo (medico)
  • Segundo descuento: 3% del valor que quedó después del primer descuento (equipo)
  • Total: $75.08

ConsultaEmergencia extends Paciente

class ConsultaEmergencia extends Paciente {

¿Cómo funciona?

  • La => es una "función flecha" (arrow function)
  • Cada rl.question() hace una pregunta y espera respuesta
  • La respuesta se guarda en la variable (nombre, fechaNacimiento, etc)
  • Luego ejecuta lo que está dentro de los {}
  • Las preguntas están anidadas (una dentro de otra)

Orden de las preguntas:

  1. Nombre del paciente
  2. Fecha de nacimiento
  3. Teléfono del paciente
  4. Nombre del responsable
  5. Teléfono del responsable

5. CÁLCULO DE EDAD

const partes = fechaNacimiento.split("/");
// Si ingresa "15/08/1995" → partes = ["15", "08", "1995"]

const fecha = new Date(parseInt(partes[2]!), parseInt(partes[1]!) - 1, parseInt(partes[0]!));
// New Date(1995, 7, 15) ← Crea la fecha de nacimiento
// Nota: Los meses van 0-11, por eso " - 1"

const hoy = new Date();
// Obtiene la fecha de hoy

let edad = hoy.getFullYear() - fecha.getFullYear();
// 2026 - 1995 = 31

const mesActual = hoy.getMonth() - fecha.getMonth();
// Si ya pasó el cumpleaños este año, edad correcta
// Si todavía no, resta 1 año

Paso a paso:

  1. split("/") divide el texto por "/" → ["15", "08", "1995"]
  2. parseInt() convierte texto a número: "1995" → 1995
  3. new Date(año, mes, día) crea un objeto fecha (los meses van 0-11)
  4. Resta el año de nacimiento del año actual
  5. Verifica si ya pasó el cumpleaños este año

CONCEPTOS CLAVE

Programación Orientada a Objetos (POO)

  • Clase: Plantilla para crear objetos (Paciente, ConsultaGeneral, etc)
  • Objeto: Una instancia de una clase (el paciente de Juan, el paciente de María)
  • Herencia: ConsultaGeneral hereda de Paciente (gets all its properties)
  • Abstracción: Clase abstracta que no se usa directamente, solo se hereda
  • Encapsulamiento: Usar protected para controlar acceso a propiedades

Arrow Functions vs Function

  • (nombre) => { } = Arrow function (la que usamos)
  • function(nombre) { } = Function tradicional (la que no estamos usando)
  • Son similares, pero arrow functions son más modernas y concisas

Tipos de Datos

  • string = texto ("Juan", "15/08/1995")
  • number = números (25, 1234567890)
  • void = el método no retorna nada
